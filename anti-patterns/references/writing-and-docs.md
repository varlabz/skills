# Writing and Documentation Anti-Patterns

## Translation Artifacts

Avoid Russian→English literal translations and over-literal phrasing:

| Anti-Pattern | Fix |
|-------------|-----|
| "and I realized..." | Delete or rephrase |
| "point." (as standalone sentence) | Delete |
| "It is worth noting that..." | Delete or state the fact directly |
| "In the modern world..." / "In today's world..." | Start with the actual topic |
| "It should be noted that..." | Delete |

## Corporate Jargon

Replace empty corporate speak with plain language:

| Anti-Pattern | Fix |
|-------------|-----|
| "leverage" | "use" or "apply" |
| "synergy" | Describe the actual collaboration |
| "paradigm shift" | Say what actually changed |
| "holistic approach" | Describe the specific approach |
| "deep dive" | "detailed look" or "analysis" |
| "move the needle" | Say what metric improved |
| "low-hanging fruit" | "easy wins" |
| "think outside the box" | Describe the actual creative approach |
| "best-in-class" | Show the evidence |
| "cutting-edge" | Name the technology |
| "At the end of the Day..." | Delete or use "ultimately" |
| "As a matter of fact..." | Delete or just state the fact |

## Vague Language

Be specific. Replace vague quantifiers with actual numbers or names:

| Anti-Pattern | Fix |
|-------------|-----|
| "significantly improved" | Give the number |
| "many people" | Give a number or source |
| "various factors" | List them |
| "quite a lot" | Be specific |
| "relatively low" | Give the actual number |
| "a number of" | Say the number |
| "things like" | Be specific |
| "etc." | List them explicitly or omit |

## Structural Problems

Write tight, well-structured content:

| Anti-Pattern | Fix |
|-------------|-----|
| Long intro before the point | State the main point in sentence 1 |
| Restating the same idea differently | Delete the repetition |
| Bullet points that aren't parallel | Make them the same grammatical form |
| Tables with one column | Use a list |
| Summary that adds nothing new | Make it actionable |
| "In conclusion..." | The last point is the conclusion |

## Hedging & Weak Language

State things directly. Remove unnecessary qualifiers:

| Anti-Pattern | Fix |
|-------------|-----|
| "might" / "could possibly" | State what you know |
| "it seems that" | State the observation |
| "I think that" | Just state it |
| "perhaps we could consider" | Make a recommendation |
| "it is important to note" | Just note it |
| "it is worth mentioning" | Just mention it — or delete |

## AI-Specific Artifacts

Never use these AI clichés:

| Anti-Pattern | Fix |
|-------------|-----|
| "Let's dive in" | Start with the content |
| "Here's the thing" | State the point |
| "Game-changer" / "game-changing" | Show the evidence |
| "Revolutionary" | Name what actually changed |
| "In today's world" | Start with the topic |
| "It's all about" | State the purpose directly |
| "At the core" | State the core directly |
| "Not just X, but Y" | Use naturally or rephrase |
| "Whether you're X or Y" | Address the audience directly |
| "In a world where..." | Start with the actual problem |
| "I'm here to" | Skip the preamble |
| "As an AI language model..." | Skip the preamble |
| "Let me break it down" | Just break it down |
| "TL;DR" used incorrectly | Use only when genuinely summarizing multiple points; place at the end |

## Overused Buzzwords

Replace with plain language:

| Anti-Pattern | Fix |
|-------------|-----|
| "delve" | "look into" or "study" |
| "tapestry" | Describe the actual relationship |
| "landscape" | Name the specific field/area |
| "testament to" | Show the evidence directly |
| "realm" | Say the actual domain |
| "paradigm" | Describe what changed |
| "ecosystem" | Describe the relationships, dependencies, and key players |
| "robust" | Say what works and how well |
| "seamless" | Describe the actual integration |
| "empower" | Say who does what |
| "unlock" | Describe the actual benefit |
| "navigate" | Describe what they actually did |
| "harness" | Describe what they actually did |

## Robotic Openings

Start with the actual topic, not formulaic intros:

| Anti-Pattern | Fix |
|-------------|-----|
| "In today's fast-paced world..." | Start with the actual topic |
| "It is important to note that..." | Just make the point |
| "One must consider..." | State what should be considered |
| "It goes without saying that..." | Skip it |
| "As we all know..." | Assume competence, skip the preamble |
| "The question of whether..." | State the question directly |

## Repetitive Transitions

Vary transitions or skip them:

| Anti-Pattern | Fix |
|-------------|-----|
| "Additionally," (every sentence) | Vary or delete |
| "Furthermore," (every paragraph) | Use naturally or skip |
| "Moreover," (every point) | Vary or delete |
| "In conclusion," | The last point is the conclusion |
| "To summarize," | Just summarize |
| "As previously mentioned" | Trust the reader remembers |
| "It is worth mentioning that" | Just mention it — or delete |

## Overly Formal Tone

Use simple words when they work:

| Anti-Pattern | Fix |
|-------------|-----|
| "Utilize" | "Use" |
| "Facilitate" | "Help" or "enable" |
| "Subsequently" | "Then" or "after" |
| "Aforementioned" | "The [thing]" or repeat the name |
| "Herein lies" | "This is" or just state it |
| "It is imperative that" | "Must" or "should" |

## Technical Documentation Anti-Patterns

Write clear, complete documentation:

| Anti-Pattern | Fix |
|-------------|-----|
| Assuming unknown prerequisites | List all prerequisites explicitly |
| Inconsistent imperative/passive voice across steps | Use consistent voice throughout |
| Missing error cases | Document common errors and how to resolve them |
| Assuming successful outcomes | Describe what to do when it doesn't work |
| Vague "may" language for deterministic behavior | Be definitive when the behavior is certain |
| Overusing "simply" or "easily" for complex operations | Describe the actual complexity |
| Writing docs that assume the reader has the same context | Assume minimal domain knowledge |

## Unnecessary Statistics & Made-Up Quotes

Don't fabricate or cite without sources:

| Anti-Pattern | Fix |
|-------------|-----|
| Statistics without context or source | Cite the source and explain relevance |
| "Studies show..." (no citation) | Name the study or skip |
| Made-up quotes attributed to real people | Use real quotes or paraphrase |
| "According to experts..." (no names) | Name the experts or skip |
| Percentages without baselines | Give the full numbers |

## Formatting Problems

Use formatting purposefully:

| Anti-Pattern | Fix |
|-------------|-----|
| Exclamation points for emphasis | Let the content carry weight |
| Bold everything | Bold only key terms |
| Overuse of em dashes — like this | Use commas or periods |
| Walls of text | Break into shorter paragraphs |
| Numbered lists with 2 items | Use bullets |
